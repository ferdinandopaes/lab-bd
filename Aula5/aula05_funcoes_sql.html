🗂 Tema da Unidade

Esta aula faz parte da Unidade 2 – Funções em SQL. Vamos entender como o MySQL oferece uma biblioteca rica de funções embutidas que resolvem problemas reais do dia a dia: formatar dados, calcular datas, agregar resultados, criar rankings e analisar séries temporais.

💡Funções são usadas em praticamente toda query profissional. Dominar esse conteúdo é o que separa quem "sabe SQL" de quem realmente trabalha bem com dados.

📐 O que é uma Função em SQL?

Uma função em SQL é um objeto nomeado que recebe zero ou mais argumentos, executa um processamento definido e retorna um único valor escalar. Analogia matemática direta: f(x) → y.

C.J. Date define função como "uma operação que mapeia um conjunto de valores de entrada em um único valor de saída". No modelo relacional, funções retornam valores atômicos — nunca uma tabela inteira.

🔍Diferença crucial: funções → valor · views → tabela · procedures → executam comandos sem retornar valor direto.

🔤 Funções de Texto (String Functions)

Operam linha por linha. Para cada linha, a função é chamada uma vez e devolve um valor.

FunçãoO que fazExemplo
UPPER(str)MaiúsculoUPPER('joao') → 'JOAO'
LOWER(str)MinúsculoLOWER('ANA') → 'ana'
CONCAT(s1,s2)ConcatenaCONCAT('Ana',' Lima')
CONCAT_WS(sep,...)Concatena com separador, ignora NULLCONCAT_WS(' ','Ana',NULL,'Lima') → 'Ana Lima'
SUBSTRING(str,pos,len)Extrai parteSUBSTRING('Fatec',1,3) → 'Fat'
TRIM(str)Remove espaços nas bordasTRIM(' dado ') → 'dado'
REPLACE(str,de,para)Substitui ocorrênciasREPLACE('2024-01-15','-','/')
LPAD(str,len,pad)Preenche à esquerdaLPAD('7',3,'0') → '007'
LENGTH(str)Comprimento em bytesLENGTH('sql') → 3
⚠️CONCAT com qualquer NULL retorna NULL! Prefira CONCAT_WS ou envolva com IFNULL.

📊 Funções de Agregação

Operam sobre um conjunto de linhas e retornam um único valor resumido. Acompanham GROUP BY ou retornam o total geral.

FunçãoComportamento com NULL
COUNT(*)Conta todas as linhas, incluindo NULLs
COUNT(coluna)Conta apenas valores não-NULL
SUM(coluna)Ignora NULLs na soma
AVG(coluna)Ignora NULLs no numerador E denominador
MAX/MIN(coluna)Funciona com texto e datas também
GROUP_CONCAT(col)Concatena todos os valores do grupo em uma string
🚨AVG ignora NULLs no numerador E denominador. AVG(nota) com NULLs ≠ SUM(nota)/COUNT(*). Isso quebra muita análise sem que ninguém perceba!

🪟 Window Functions (MySQL 8.0+)

Calculam sobre um conjunto de linhas relacionadas à linha atual sem colapsá-las. Diferença fundamental em relação à agregação: você mantém todas as linhas no resultado.

Sintaxe base
função() OVER (
    PARTITION BY coluna_agrupamento
    ORDER BY    coluna_ordenação
    ROWS BETWEEN 1 PRECEDING AND CURRENT ROW
)
FunçãoPropósito
ROW_NUMBER()Número sequencial único por partição
RANK()Posição com empates — pula números
DENSE_RANK()Posição com empates — não pula
LAG(col, n)Acessa linha n posições atrás
LEAD(col, n)Acessa linha n posições à frente
SUM() OVER()Soma acumulada ou por partição

🪤 Armadilhas Clássicas

Erro 1 — Função sobre coluna indexada no WHERE

Performance — Índice x Função
-- ERRADO: full table scan, ignora o índice
WHERE YEAR(data_pedido) = 2024

-- CERTO: aproveita o índice da coluna
WHERE data_pedido BETWEEN '2024-01-01' AND '2024-12-31'

Erro 2 — Confundir WHERE com HAVING

WHERE vs HAVING
-- ERRADO: HAVING não filtra colunas puras
SELECT depto, COUNT(*) FROM funcionarios
HAVING nome LIKE 'A%'

-- CERTO: WHERE filtra ANTES; HAVING filtra APÓS a agregação
SELECT depto, COUNT(*) FROM funcionarios
WHERE  nome LIKE 'A%'
GROUP BY depto
HAVING COUNT(*) > 3

Erro 3 — CONCAT com NULL

NULL em CONCAT
-- ERRADO: sobrenome NULL contamina todo o resultado
SELECT CONCAT(nome, ' ', sobrenome) FROM clientes;

-- CERTO: CONCAT_WS ignora NULLs automaticamente
SELECT CONCAT_WS(' ', nome, sobrenome) FROM clientes;

Laboratório — TechVenda Ltda.

Você é analista de banco de dados de um distribuidor de eletrônicos. Execute os scripts em sequência no MySQL Workbench.

🏗 Script 1 — Criação do Banco e Tabelas

01_criar_banco.sql
-- ================================================
-- LABORATÓRIO 05 — FUNÇÕES EM SQL
-- TechVenda Ltda. | Profº Ferdinando Paes
-- ================================================
CREATE DATABASE IF NOT EXISTS techvenda;
USE techvenda;

CREATE TABLE IF NOT EXISTS categorias (
    id_categoria INT AUTO_INCREMENT PRIMARY KEY,
    nome         VARCHAR(50) NOT NULL
);

CREATE TABLE IF NOT EXISTS produtos (
    id_produto   INT AUTO_INCREMENT PRIMARY KEY,
    nome         VARCHAR(100) NOT NULL,
    preco        DECIMAL(10,2) NOT NULL,
    estoque      INT DEFAULT 0,
    id_categoria INT,
    FOREIGN KEY (id_categoria) REFERENCES categorias(id_categoria)
);

CREATE TABLE IF NOT EXISTS clientes (
    id_cliente    INT AUTO_INCREMENT PRIMARY KEY,
    nome          VARCHAR(80) NOT NULL,
    sobrenome     VARCHAR(80),
    email         VARCHAR(120),
    dt_nascimento DATE,
    cidade        VARCHAR(60),
    estado        CHAR(2)
);

CREATE TABLE IF NOT EXISTS pedidos (
    id_pedido  INT AUTO_INCREMENT PRIMARY KEY,
    id_cliente INT NOT NULL,
    dt_pedido  DATETIME DEFAULT NOW(),
    status     ENUM('Pendente','Aprovado','Cancelado','Entregue'),
    FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente)
);

CREATE TABLE IF NOT EXISTS itens_pedido (
    id_item    INT AUTO_INCREMENT PRIMARY KEY,
    id_pedido  INT NOT NULL,
    id_produto INT NOT NULL,
    quantidade INT NOT NULL,
    preco_unit DECIMAL(10,2) NOT NULL,
    FOREIGN KEY (id_pedido)  REFERENCES pedidos(id_pedido),
    FOREIGN KEY (id_produto) REFERENCES produtos(id_produto)
);

🔤 Script 3 — Funções de Texto

03_funcoes_texto.sql
-- Nome completo em maiúsculo
SELECT
    UPPER(CONCAT_WS(' ', nome, sobrenome)) AS nome_maiusculo,
    LOWER(email)                           AS email_normalizado
FROM clientes;

-- Máscara de e-mail
SELECT
    nome,
    CONCAT(LEFT(email,3), '***@',
           SUBSTRING_INDEX(email,'@',-1)) AS email_mascarado
FROM clientes
WHERE email IS NOT NULL;

-- Código do produto formatado com 5 dígitos
SELECT
    nome,
    LENGTH(nome)             AS tamanho,
    LEFT(nome, 15)           AS nome_curto,
    LPAD(id_produto, 5, '0') AS codigo_formatado
FROM produtos;

📅 Script 4 — Funções de Data

04_funcoes_data.sql
-- Idade calculada corretamente
SELECT
    CONCAT_WS(' ', nome, sobrenome)                  AS cliente,
    TIMESTAMPDIFF(YEAR, dt_nascimento, CURDATE())   AS idade_anos
FROM clientes
ORDER BY idade_anos DESC;

-- Relatório mensal de pedidos
SELECT
    DATE_FORMAT(dt_pedido, '%m/%Y')                  AS mes_ano,
    COUNT(*)                                          AS total_pedidos,
    SUM(CASE WHEN status='Entregue'  THEN 1 ELSE 0 END) AS entregues,
    SUM(CASE WHEN status='Cancelado' THEN 1 ELSE 0 END) AS cancelados
FROM pedidos
GROUP BY YEAR(dt_pedido), MONTH(dt_pedido),
         DATE_FORMAT(dt_pedido,'%m/%Y')
ORDER BY YEAR(dt_pedido), MONTH(dt_pedido);

📊 Script 5 — Agregação com JOIN e HAVING

05_agregacao.sql
-- Receita por categoria — apenas acima de R$ 5.000
SELECT
    c.nome                                      AS categoria,
    COUNT(DISTINCT p.id_pedido)               AS total_pedidos,
    SUM(ip.quantidade * ip.preco_unit)         AS receita_bruta,
    ROUND(AVG(ip.preco_unit), 2)              AS ticket_medio
FROM itens_pedido ip
JOIN pedidos    p  ON p.id_pedido    = ip.id_pedido
JOIN produtos   pr ON pr.id_produto  = ip.id_produto
JOIN categorias c  ON c.id_categoria = pr.id_categoria
WHERE p.status <> 'Cancelado'
GROUP BY c.id_categoria, c.nome
HAVING receita_bruta > 5000
ORDER BY receita_bruta DESC;

🪟 Script 6 — Window Functions (MySQL 8+)

06_window_functions.sql
-- Ranking de produtos por receita
SELECT
    pr.nome                                      AS produto,
    SUM(ip.quantidade * ip.preco_unit)           AS receita,
    ROW_NUMBER() OVER (
        ORDER BY SUM(ip.quantidade * ip.preco_unit) DESC
    ) AS posicao,
    DENSE_RANK() OVER (
        ORDER BY SUM(ip.quantidade * ip.preco_unit) DESC
    ) AS ranking
FROM itens_pedido ip
JOIN produtos pr ON pr.id_produto = ip.id_produto
GROUP BY pr.id_produto, pr.nome
ORDER BY receita DESC;

-- LAG: variação mensal de receita
SELECT
    mes_ano,
    receita,
    LAG(receita, 1, 0) OVER (ORDER BY ano, mes)  AS mes_anterior,
    ROUND(
        (receita - LAG(receita,1,0) OVER(ORDER BY ano,mes))
        / NULLIF(LAG(receita,1,0) OVER(ORDER BY ano,mes),0)*100, 1
    ) AS variacao_pct
FROM (
    SELECT
        YEAR(p.dt_pedido)                  AS ano,
        MONTH(p.dt_pedido)                 AS mes,
        DATE_FORMAT(p.dt_pedido,'%m/%Y') AS mes_ano,
        SUM(ip.quantidade*ip.preco_unit)   AS receita
    FROM pedidos p
    JOIN itens_pedido ip ON ip.id_pedido = p.id_pedido
    WHERE p.status <> 'Cancelado'
    GROUP BY ano, mes, mes_ano
) AS resumo
ORDER BY ano, mes;
⚠️Verifique a versão antes de executar: SELECT VERSION(); — Window Functions exigem MySQL 8.0+.
Exercícios de Fixação

🟢 Básicos (1–5)

BásicoEx. 01 — Nome completo formatado▼
Liste todos os clientes exibindo o nome completo (usando CONCAT_WS), cidade e estado formatados como "Ana Silva — Garça/SP". Use concatenação e UPPER no estado.
BásicoEx. 02 — Preço em formato brasileiro▼
Mostre o nome de todos os produtos com o preço formatado em reais: R$ 8.999,90. Use FORMAT, CONCAT e REPLACE para adaptar o separador ao padrão PT-BR.
BásicoEx. 03 — Idade dos clientes▼
Calcule a idade de cada cliente em anos usando TIMESTAMPDIFF. Exiba somente clientes com mais de 30 anos, do mais velho para o mais novo.
BásicoEx. 04 — Classificação de estoque▼
Exiba todos os produtos com situação de estoque: 'Crítico' (0), 'Baixo' (1–20), 'Normal' (21–100) e 'Alto' (acima de 100). Use CASE WHEN.
BásicoEx. 05 — COUNT(*) vs COUNT(coluna)▼
Conte o total de clientes com e-mail cadastrado e o total sem e-mail (NULL). Use COUNT(email) e COUNT(*) na mesma query para ver a diferença na prática.

🔵 Intermediários (6–10)

IntermediárioEx. 06 — Relatório de pedidos com urgência▼
Crie um relatório com: código do pedido com 6 dígitos (LPAD), data em dd/mm/aaaa hh:mm, dias desde o pedido (DATEDIFF) e coluna urgencia: 'ATRASADO' se mais de 60 dias, 'NORMAL' caso contrário. Use IF.
IntermediárioEx. 07 — Desempenho por produto▼
Gere relatório com: nome do produto, quantidade total vendida, receita total, preço médio de venda e diferença entre o preço médio de venda e o preço de tabela. Filtre produtos com mais de 1 unidade vendida usando HAVING.
IntermediárioEx. 08 — Receita mensal por categoria▼
Liste a receita mensal por categoria (mês/ano + categoria + receita), exibindo apenas meses em que a categoria gerou mais de R$ 1.000,00. Exclua pedidos cancelados. Use DATE_FORMAT, GROUP BY e HAVING.
IntermediárioEx. 09 — Histórico de compras▼
Exiba nome completo de cada cliente, data do primeiro pedido (MIN), data do último (MAX) e número de dias entre eles (DATEDIFF). Apenas clientes com ao menos 1 pedido 'Entregue'.
IntermediárioEx. 10 — Lista de produtos por cliente▼
Usando GROUP_CONCAT, gere uma linha por cliente com todos os produtos comprados separados por " → ", ordenados alfabeticamente. Exclua pedidos cancelados.

🟣 Desafiadores (11–12)

DesafioEx. 11 — Dashboard executivo em uma query▼
Uma única query que retorne, para cada mês: mês/ano, total de pedidos, total entregues, receita bruta, receita média por pedido entregue e classificação do mês: 'Ótimo' (receita > R$ 10.000), 'Bom' (> R$ 5.000), 'Regular' (> R$ 1.000) ou 'Fraco'. Ordene cronologicamente.
DesafioEx. 12 — Ranking com variação mensal (Window)▼
Usando Window Functions: nome do produto, receita do mês atual, receita do mês anterior (LAG), variação percentual, posição no ranking (DENSE_RANK) e coluna com 'CRESCIMENTO' ou 'QUEDA'. Apenas produtos vendidos em pelo menos 2 meses diferentes.
