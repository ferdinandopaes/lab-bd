📦 Parte 1 — 25 min

List vs MutableList

A primeira decisão ao criar qualquer coleção em Kotlin: ela precisa mudar? Se não — use List. Se sim — use MutableList.

⚖️ Imutável vs Mutável

List — Imutável
val pratos: List<String> =
    listOf("Filé", "Risoto")

// pratos.add("x") → ERRO!
// pratos.removeAt(0) → ERRO!

// Leitura OK:
println(pratos[0]) // Filé
println(pratos.size) // 2
MutableList — Mutável
val lista: MutableList<String> =
    mutableListOf("Filé")

lista.add("Risoto")   // ✅
lista.removeAt(0)    // ✅
lista.set(0, "Salmão") // ✅

println(lista.size)    // 1
✅ Regra prática: Prefira sempre List (imutável). Use MutableList apenas quando realmente precisar adicionar ou remover itens em tempo de execução.

🔍 Acessando Elementos

Kotlin 
val frutas = listOf("Maçã", "Banana", "Laranja")

// Acesso por índice (começa em 0)
val primeira  = frutas[0]          // "Maçã"
val segunda   = frutas[1]          // "Banana"

// Funções mais expressivas — prefira estas!
val prim2   = frutas.first()      // "Maçã"
val ultima  = frutas.last()       // "Laranja"
val tam     = frutas.size         // 3
val vazia   = frutas.isEmpty()   // false

// Iteração com índice
frutas.forEachIndexed { index, item ->
    println("$index: $item")
}
// 0: Maçã
// 1: Banana
// 2: Laranja

🧯 Acesso Seguro — Evite IndexOutOfBoundsException

Kotlin 
val lista = listOf("A", "B")

// Perigoso — lança exceção se índice inválido
val x = lista[5]   // 💥 IndexOutOfBoundsException

// Seguro — retorna null se não existir
val y = lista.getOrNull(5)   // null (sem crash)
val z = lista.getOrElse(5) { "Padrão" }  // "Padrão"
⚠️ Prefira getOrNull ou getOrElse em vez de acesso direto por índice quando o índice pode ser inválido — evita crashes em produção.
⚡ Parte 2 — 35 min

Funções de Alta Ordem em Listas

O coração do estilo funcional em Kotlin. Funções que recebem outra função como parâmetro e transformam suas coleções com elegância.

🗂 Referência das Funções

Clique em cada card para ver a explicação e o exemplo completo.

forEach
Lista → Unit (percorrer)
Percorre todos os elementos. Não retorna nada — use quando o objetivo é apenas executar uma ação para cada item.
Kotlin
// Forma explícita
pratos.forEach { prato ->
    println(prato.nome)
}

// Forma curta com 'it'
pratos.forEach { println(it.nome) }
filter
Lista → Lista (filtrada)
Retorna uma NOVA lista contendo apenas os elementos que passam na condição (predicado). A lista original não é alterada.
Kotlin
val entradas = pratos
    .filter { it.categoria == "Entrada" }

val caros = pratos
    .filter { it.preco > 50.0 }

val busca = pratos.filter {
    it.nome.contains("Filé", ignoreCase = true)
}
map
Lista<A> → Lista<B> (transformar)
Transforma cada elemento da lista em outro valor. O tipo de retorno PODE ser diferente do tipo de entrada.
Kotlin
// List<Prato> → List<String>
val nomes = pratos.map { it.nome }

// List<Prato> → List<Double>
val precos = pratos.map { it.preco }

// Aplica 10% de desconto em todos
val comDesconto = pratos.map {
    it.copy(preco = it.preco * 0.9)
}
sumOf
Lista → Number (somar campo)
Soma um campo numérico de todos os elementos. A forma mais direta de calcular totais em coleções de objetos.
Kotlin
val totalCardapio = pratos
    .sumOf { it.preco }

val totalEntradas = pratos
    .filter { it.categoria == "Entrada" }
    .sumOf { it.preco }
groupBy
Lista → Map<K, List<V>>
Agrupa os elementos por uma chave. Retorna um Map onde cada chave aponta para uma lista de elementos com aquela chave.
Kotlin
val porCategoria:
    Map<String, List<Prato>> =
        pratos.groupBy { it.categoria }

// Acessar um grupo específico
porCategoria["Entrada"]
    ?.forEach { println(it.nome) }
sortedBy / maxByOrNull
Lista → Lista ordenada / elemento
sortedBy retorna a lista ordenada por um critério. maxByOrNull retorna o elemento com o maior valor de um campo (ou null se vazia).
Kotlin
// Ordenar por preço (crescente)
val porPreco = pratos
    .sortedBy { it.preco }

// Prato mais caro (nullable!)
val maisCaro = pratos
    .maxByOrNull { it.preco }

// Preço médio
val media = pratos
    .map { it.preco }
    .average()

🔗 Chaining — Encadeando Funções

A magia real acontece quando você encadeia funções. Cada função retorna uma nova lista que pode ser processada pela próxima.

Kotlin — Chain de operações 
// Nomes dos pratos principais ordenados por preço
val resultado = pratos
    .filter  { it.categoria == "Prato Principal" }
    .sortedBy { it.preco }
    .map    { "${it.nome} — R$ ${it.preco}" }

resultado.forEach { println(it) }

// Contagem de pratos por categoria
pratos
    .groupBy { it.categoria }
    .forEach { (cat, lista) ->
        println("$cat: ${lista.size} pratos")
    }
💡 Cada função no chain não modifica a lista original — ela retorna uma nova. A lista pratos continua intacta ao longo de todo o chain.

📊 Estatísticas no Logcat

No MainActivity.kt, dentro do onCreate, adicione este bloco para verificar os dados.

MainActivity.kt — onCreate 
val TAG = "CardapioStats"

// 1. Quantidade total
Log.d(TAG, "Total de pratos: ${cardapioCompleto.size}")

// 2. Preço médio
val media = cardapioCompleto.map { it.preco }.average()
Log.d(TAG, "Preço médio: R$ %.2f".format(media))

// 3. Mais caro e mais barato
val maisCaro   = cardapioCompleto.maxByOrNull { it.preco }
val maisBarato = cardapioCompleto.minByOrNull { it.preco }
Log.d(TAG, "Mais caro: ${maisCaro?.nome} — R$ ${maisCaro?.preco}")
Log.d(TAG, "Mais barato: ${maisBarato?.nome} — R$ ${maisBarato?.preco}")

// 4. Contagem por categoria
cardapioCompleto
    .groupBy { it.categoria }
    .forEach { (categoria, lista) ->
        Log.d(TAG, "$categoria: ${lista.size} pratos")
    }

// 5. Pratos acima de R$ 40
val premium = cardapioCompleto
    .filter { it.preco > 40.0 }
Log.d(TAG, "Pratos acima de R$ 40:")
premium.forEach { Log.d(TAG, "  - ${it.nome}: R$ ${it.preco}") }
✅ Filtre o Logcat com a tag CardapioStats para ver apenas as suas mensagens. Use o campo de busca no Android Studio.
