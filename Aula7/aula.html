Parte 1 · 15 min
Anatomia do SELECT
A estrutura fundamental de toda consulta SQL
1
Estrutura completa — analogia: o bibliotecário
SQL
SELECT   colunas           -- O QUE trazer
FROM     tabela            -- ONDE buscar
WHERE    condição          -- FILTRO (opcional)
ORDER BY coluna [ASC|DESC] -- ORDEM (opcional)
LIMIT    N;                -- QUANTIDADE (opcional)

⚡ Ordem de execução interna

O banco não executa na ordem que você escreve:

1FROM
→
2WHERE
→
3SELECT
→
4ORDER BY
→
5LIMIT
Bug clássico: WHERE alias = 'x' falha porque o alias do SELECT ainda não existe quando o WHERE executa.
Parte 2 · 35 min
WHERE
Comparação
O porteiro que decide quem entra
2
Igual / Diferente / Range
SELECT * FROM clientes
WHERE estado = 'SP';

SELECT * FROM produtos
WHERE categoria <> 'Periférico';

SELECT * FROM produtos
WHERE preco_venda > 500;

SELECT * FROM produtos
WHERE preco_custo
  BETWEEN 100 AND 500;
BETWEEN com datas
SELECT * FROM pedidos
WHERE data_pedido
  BETWEEN '2024-01-01'
      AND '2024-12-31';
BETWEEN é inclusivo — inclui os dois extremos do intervalo.
= igual <> diferente BETWEEN range
Parte 2 · continuação
LIKE
O Detetive
de Padrões
% = qualquer texto
_ = 1 caractere
2
Padrões de busca
-- Começa com 'Ana'
WHERE nome LIKE 'Ana%';

-- Termina com 'Silva'
WHERE nome LIKE '%Silva';

-- Contém 'Paulo'
WHERE cidade LIKE '%Paulo%';

-- Estado com 2 chars terminando em P
WHERE estado LIKE '_P';
Resolvendo o problema da abertura
-- Detecta qualquer grafia de Garça
SELECT COUNT(*)
FROM clientes
WHERE UPPER(TRIM(cidade))
  IN ('GARÇA', 'GARCA');
⚠️ LIKE '%texto%' é lento em tabelas grandes — o banco precisa varrer todos os registros. Em produção com milhões de linhas, pode travar o sistema.
Parte 2 · continuação
Operadores
Lógicos
AND · OR · IN · NOT IN
2
AND · OR · IN
-- AND: todas verdadeiras
WHERE status = 'entregue'
  AND frete > 20;

-- OR: pelo menos uma verdadeira
WHERE estado = 'SP'
   OR estado = 'RJ';

-- IN: mais limpo que múltiplos OR
WHERE estado IN ('SP','RJ','MG');

-- NOT IN: excluir lista
WHERE status NOT IN ('cancelado','pendente');
Precedência — use parênteses!
-- ❌ Resultado inesperado
WHERE estado = 'SP'
   OR estado = 'RJ'
  AND cidade LIKE '%Paulo%';

-- ✅ Intenção clara com parênteses
WHERE (estado = 'SP'
    OR estado = 'RJ')
  AND cidade LIKE '%Paulo%';
AND tem precedência sobre OR. Sem parênteses, o resultado pode ser completamente diferente do que você espera.
⚠️
NULL
não é zero.
NULL é a ausência de informação. É como perguntar a idade de alguém que preferiu não informar — a resposta não é zero, é "não sei".
NULL ≠ 0  ·  NULL ≠ ''  ·  NULL ≠ 'NULL'
Como usar corretamente
-- ✅ CORRETO
SELECT * FROM pedidos
WHERE data_entrega IS NULL;

SELECT * FROM clientes
WHERE email IS NOT NULL;
-- ❌ NUNCA funciona — retorna 0 linhas
SELECT * FROM pedidos
WHERE data_entrega = NULL;
Por que? O operador = compara valores. NULL não é um valor — é a ausência dele. NULL = NULL não retorna TRUE, retorna NULL.
Parte 3 · 15 min
ORDER BY
e LIMIT
Classificar, top-N e paginação
3
Ordenação simples e múltipla
-- Mais caros primeiro
SELECT nome, preco_venda
FROM produtos
ORDER BY preco_venda DESC;

-- Por estado e depois por nome
SELECT nome, cidade, estado
FROM clientes
ORDER BY estado ASC, nome ASC;
Top-N e paginação
-- Top 3 produtos mais caros
SELECT nome, preco_venda
FROM produtos
ORDER BY preco_venda DESC
LIMIT 3;

-- Página 3 (5 por página)
SELECT nome, cidade
FROM clientes
ORDER BY nome ASC
LIMIT 5 OFFSET 10;
APIs REST usam LIMIT/OFFSET por baixo dos panos. ?page=3&limit=20 → LIMIT 20 OFFSET 40

Funções
de String
Sua defesa contra dados sujos
UPPER · LOWER · TRIM
REPLACE · CONCAT · LENGTH
4
Padronizar caixa e limpar espaços
SELECT
  UPPER(cidade),
  LOWER(estado),
  TRIM(categoria),
  LENGTH(TRIM(categoria))

Substituir e concatenar
-- Corrigir grafia sem alterar o banco
SELECT
  REPLACE(cidade,'Garca','Garça')
FROM clientes;

-- Montar rótulo completo
SELECT
  CONCAT(nome, ' — ',
    cidade, '/', estado) AS local
FROM clientes;
Detectar o espaço invisível da Aula 05:  WHERE LENGTH(categoria) <> LENGTH(TRIM(categoria)) — encontra a 'Mobiliário ' com espaço no final.

Funções
de Data
Extrair, calcular e filtrar por período
4
Extrair partes da data
SELECT NOW(), CURDATE(), CURTIME();

SELECT
  id_pedido,
  YEAR(data_pedido)    AS ano,

Calcular prazo e dias em aberto
-- Prazo de entrega em dias
SELECT
  id_pedido,
  DATEDIFF(data_entrega, data_pedido)
    AS dias_para_entregar
FROM pedidos
WHERE data_entrega IS NOT NULL
ORDER BY dias_para_entregar DESC;

-- Pedidos abertos há mais de 30 dias
SELECT id_pedido, status,
  DATEDIFF(CURDATE(), data_pedido)
    AS dias_aberto
FROM pedidos
WHERE status IN ('pendente','enviado')
  AND DATEDIFF(CURDATE(),data_pedido)>30;
Parte 4 · continuação
Funções
Numéricas
Calcular indicadores direto no SQL
4
Margem de lucro por produto
SELECT
  nome,
  preco_custo,
  preco_venda,
  ROUND(
    (preco_venda - preco_custo) / preco_custo * 100, 2
  ) AS margem_pct
FROM produtos
ORDER BY margem_pct DESC;

ROUND(n, d)

Arredonda n para d casas decimais. Crítico em sistemas financeiros.

CEIL / FLOOR

CEIL(4.1) = 5 (para cima)
FLOOR(4.9) = 4 (para baixo)

ABS(n)

Valor absoluto. Remove o sinal negativo. Útil para diferenças.

Pergunta 1 — Logística
"Quais pedidos ainda não foram entregues?"
SELECT
  id_pedido, id_cliente,
  data_pedido, status,
  DATEDIFF(CURDATE(),
    data_pedido) AS dias_aguardando
FROM pedidos
WHERE status IN ('pendente', 'enviado')
ORDER BY data_pedido ASC;
Pergunta 2 — CFO
"Quais produtos têm margem acima de 80%?"
SELECT
  nome,
  ROUND((preco_venda - preco_custo)
    / preco_custo * 100, 1)
    AS margem_pct
FROM produtos
WHERE
  (preco_venda - preco_custo)
  / preco_custo > 0.80
ORDER BY margem_pct DESC;

Pergunta 3 — Marketing
"Clientes de SP sem email — campanha perdida?"
SELECT
  nome,
  cidade,
  telefone
FROM clientes
WHERE estado = 'SP'
  AND email IS NULL
ORDER BY nome;
Pergunta 4 — Logística
"Quais pedidos têm frete acima da média?"
SELECT
  id_pedido,
  frete,
  ROUND(
    (SELECT AVG(frete)
     FROM pedidos), 2)
  AS media_frete
FROM pedidos
WHERE frete >
  (SELECT AVG(frete)
   FROM pedidos)
ORDER BY frete DESC;
