Fundamentação Teórica

Nesta seção, exploramos a lógica relacional por trás da junção de tabelas. Compreender o "porquê" é essencial antes de escrever o código.

O que é o INNER JOIN?

Em um banco de dados relacional, os dados são normalizados para evitar redundância. O INNER JOIN é o mecanismo que nos permite "recolar" esses dados no momento da consulta, baseando-se em uma coluna comum (geralmente PK e FK).

Segundo Elmasri & Navathe, retorna apenas as linhas onde há uma correspondência em ambas as tabelas.

❌ Erros Comuns

  • Esquecer o ON: Gera Produto Cartesiano e trava o servidor.
  • Ambiguidade: Usar colunas com mesmo nome sem especificar a tabela.
  • Confusão com WHERE: Fazer a junção no WHERE é padrão obsoleto (SQL-89).

✅ Boas Práticas

  • Sempre use Aliases: Ex: FROM Funcionarios AS f.
  • Selecione apenas o necessário: Evite SELECT * para não trazer chaves repetidas.

Parte Prática – Laboratório

Aplique os conceitos criando um banco de dados real. Use os scripts abaixo no seu MySQL Workbench.

Passo 1: Preparação do Ambiente

CREATE DATABASE fatec_lab_join; USE fatec_lab_join; CREATE TABLE categorias ( id_categoria INT PRIMARY KEY AUTO_INCREMENT, nome_categoria VARCHAR(50) NOT NULL ); CREATE TABLE produtos ( id_produto INT PRIMARY KEY AUTO_INCREMENT, nome_produto VARCHAR(100) NOT NULL, preco DECIMAL(10,2), fk_categoria INT, CONSTRAINT fk_prod_cat FOREIGN KEY (fk_categoria) REFERENCES categorias(id_categoria) ); -- Inserção de Dados INSERT INTO categorias (nome_categoria) VALUES ('Eletrônicos'), ('Móveis'), ('Limpeza'); INSERT INTO produtos (nome_produto, preco, fk_categoria) VALUES ('Smartphone', 1500.00, 1), ('Notebook', 3500.00, 1), ('Sofá 3 lugares', 1200.00, 2);

Passo 2: INNER JOIN Simples (2 Tabelas)

Objetivo: Listar produtos e os nomes de suas categorias.

SELECT p.nome_produto, p.preco, c.nome_categoria FROM produtos AS p INNER JOIN categorias AS c ON p.fk_categoria = c.id_categoria;

🟢 Nível Básico (Aquecimento)

Dica: Use funções de agregação (COUNT) junto com GROUP BY e INNER JOIN.

🟡 Nível Intermediário (Raciocínio)

1. Crie uma consulta que retorne o nome do cliente, o ID do pedido e a data, mas apenas para clientes com domínio "@fatec.sp.gov.br".

2. Se inserirmos uma categoria "Jardinagem" sem produtos, ela aparecerá em um INNER JOIN com a tabela produtos?

Não. O INNER JOIN exige correspondência nas duas tabelas. Como não há produtos com o ID de Jardinagem, a categoria é omitida do resultado. (Para forçar o aparecimento, usaríamos LEFT JOIN).
