📦 A Bagunça da Fatec Express

A Fatec Express é uma empresa de logística e vendas que cresceu rápido demais. Sua gestão de dados ficou para trás e, atualmente, toda a operação depende de três arquivos Excel isolados: clientes.xlsx, pedidos.xlsx e entregas.xlsx.

Consequências Atuais (Dores do Cliente):

Redundância

O mesmo cliente é cadastrado várias vezes com nomes levemente diferentes, poluindo a base.

Inconsistência

O estoque diz que tem o produto, mas a planilha de vendas não foi atualizada. Dados não batem.

Decisões no Chute

Sem cruzamento de dados, a diretoria não sabe qual região é mais lucrativa ou onde estão os gargalos.

Planilha vs Banco de Dados

Por que o Excel não serve mais? Analise a comparação estrutural entre as duas abordagens para convencer a diretoria.

Comparativo de Capacidade

Nota: Eficiência refere-se à ausência de redundância.

Integridade SGBD Vence

Planilhas aceitam qualquer dado em qualquer célula. SGBDs usam restrições (Constraints) e tipagem rigorosa para garantir a qualidade da informação.

Volume SGBD Vence

O Excel trava ao se aproximar de 1 milhão de linhas. Um SGBD como o MySQL lida com milhões ou bilhões de registros de forma otimizada.

Concorrência SGBD Vence

Planilhas geram conflito com múltiplos usuários. Bancos de dados são desenhados para dezenas de conexões simultâneas nativamente.

Modelagem Relacional e Cardinalidade

Transformando os requisitos em um Diagrama Entidade-Relacionamento (DER). O papel do Arquiteto de Dados.

Regras de Cardinalidade (O Segredo do Relacional)

1:N (Um para Muitos)

Um Cliente faz muitos Pedidos. A Chave Estrangeira (FK) sempre migra para a tabela do lado "Muitos" (Pedidos recebe o id_cliente).

N:M (Muitos para Muitos)

Um Pedido tem muitos Produtos e um Produto está em muitos Pedidos. Atenção: Isso exige a criação de uma Tabela Associativa!

Prática Guiada

Criação do Modelo Lógico

  • 1 Abra o MySQL Workbench e vá em File → New Model.
  • 2 Adicione um diagrama clicando em Add Diagram.
  • 3 Crie as 5 tabelas discutidas na modelagem com os tipos de dados corretos (INT para IDs, VARCHAR para textos, DECIMAL(10,2) para valores).
  • 4 Conecte as tabelas usando a ferramenta de relacionamento 1:N. Lembre-se que a FK fica do lado do 'N'.
