1. Fundamentação Teórica

Antes de iniciarmos a construção do banco da IBM, é crucial distinguir a linguagem da ferramenta e entender os pilares da manipulação de dados.

🗣️

SQL vs. MySQL

SQL é o padrão (regras de trânsito).
MySQL é o motor (ferramenta) que executa as regras.

🏗️

DDL (Estrutura)

Define o esqueleto.
CREATE, ALTER, DROP.
Lida com metadados.

📝

DML (Dados)

Manipula o conteúdo.
INSERT, UPDATE, DELETE, SELECT.

⚠️

Boas Práticas

Use Notepad++ para rascunhos. Nunca esqueça o WHERE. Respeite as Chaves (PK/FK).

3. Exercícios de Fixação

Valide seu aprendizado. Clique para ver os detalhes.

1 Nível Básico ▼
  • Crie tabela projetos (id, nome, orcamento).
  • Insira 2 departamentos extras.
  • Altere projetos adicionando data_inicio.
  • Selecione funcionários ganhando > R$ 5.000.
  • Remova coluna localizacao de departamentos.
2 Nível Intermediário ▼
  • Fechamento Unidade 'Vendas': Transfira 'Ricardo Lima' para Depto 10, depois DELETE Depto 30.
  • Aumento de 15% para admitidos antes de 2022.
  • Selecione nome/cargo do Depto 20 ordenado por nome.
  • Adicione restrição CHECK para salário > R$ 1.412.
  • Liste departamentos começando com 'S'.
3 Desafio Extra (Projeto) ▼

Cenário: Alocação de Projetos

  • Crie tabela associativa alocacao_projetos (id_func, id_projeto, horas).
  • Use Chave Primária Composta para evitar duplicidade.
  • Relacione quais funcionários estão em projetos com orçamento > 100k.
